# 📋 Hexly-Boards
[![Maintainability](https://qlty.sh/gh/Hardtmuth/projects/hexly_edu_practice_2_3cource/maintainability.svg)](https://qlty.sh/gh/Hardtmuth/projects/hexly_edu_practice_2_3cource)

Интерактивное веб-приложение (SPA) для управления проектами и задачами в стиле Канбан-доски. Проект разработан в рамках производственной практики и представляет собой клиент-серверное решение с разделением логики на фронтенд и бэкенд, включает в себя полноценную систему авторизации пользователей, интерактивный интерфейс перетаскивания задач (Drag & Drop), а также контекстное меню для гибкого управления структурой колонок.  

Каталог предложенных проектов для практики: https://github.com/practical-tutorials/project-based-learning  
Выбранная ссылка из каталога: [Create a Trello Clone](http://codeloveandboards.com/blog/2016/01/04/trello-tribute-with-phoenix-and-react-pt-1/)  

Демо: https://hexly-boards.atkom.mine.nu:1443  

## 🛠 Стек технологий

**Frontend**: React, Mantine, Redux Toolkit, Vite, i18next.  
**Backend**: Node.js, Fastify, JWT, Bcrypt.  
**Database** / Infrastructure: PostgreSQL, пул соединений pg.Pool, Docker Compose.  
**Инструменты качества кода**: ESLint.

## ⚙️ Настройка переменных окружения (.env)
Для корректной работы пула соединений PostgreSQL и бэкенда, создайте файл .env в директории ./backend/docker/ (или скопируйте значения из примера):
```sh
PG_DB=hexly-board
PG_DATA=/var/lib/postgresql/data
PG_USER=hexly
PG_PASS=password
PG_PORT=5436
JWT_SECRET=your_super_secret_jwt_key
```

## 🚀 Быстрый запуск (Режим разработки)
Все основные команды автоматизированы с помощью Makefile.

> Важно: Для фонового запуска процессов frontend и backend используются утилиты nohup и pkill. Логи сервера и клиента записываются в директорию ./log/. Перед запуском убедитесь, что папка ./log создана.

### Инструкция по запуску:
1. Клонируйте репозиторий и установите зависимости:
```sh
npm install
```
2. Настройте переменные окружения, как описано в разделе выше.

Запустите всю среду разработки: Команда автоматически поднимет базу данных в Docker, запустит бэкенд и фронтенд в фоновом режиме:
```sh
make run-dev
```

> После успешного запуска логи работы приложений можно найти в файлах ./log/backend.log и ./log/frontend.log.

3. Остановка среды разработки: Команда остановит контейнер с базой данных и корректно завершит фоновые процессы Node.js и Vite:
```sh
make stop-dev
```

### Также предусмотрено автономное управление Базой Данных
Вы можете управлять Docker-контейнером PostgreSQL независимо от запуска серверов:
- **Запуск только СУБД:** `make start-db`
- **Остановка только СУБД:** `make stop-db`
- **Полная очистка данных БД (удаление папки volumes):** `make remove-db-data`

Если вам необходимо запустить сервер или клиент в интерактивном режиме (чтобы видеть логи в реальном времени прямо в текущем терминале):
- **Запуск фронтенда Vite:** `make start-frontend`
- **Запуск бэкенда Fastify:** `make start-backend`


Для проверки кода или автоматического исправления ошибок линтера с игнорированием директории Docker-данных используйте команды:
- **Только проверка:** `make lint`
- **Автоматическое исправление ошибок:** `make lint-fix`

## 🌐 Спецификация REST API (v1)
Все эндпоинты бэкенда используют базовый префикс /api/v1. Логика работы с базой данных изолирована в слое данных (queries.js).
||||
|-|-|-|
| GET    | /                           | Проверка работоспособности сервера ({"hello": "world"}) |
| POST   | /api/v1/auth/register       | Регистрация нового пользователя. Хэширует пароль через bcrypt и возвращает авторизационный JWT-токен |
| POST   | /api/v1/auth/login          | Аутентификация пользователя. Проверяет корректность данных и возвращает токен доступа (expiresIn: '24h') |
| PUT    | /api/v1/auth/update-account | Динамическое обновление разрешенных колонок профиля (name, email, password) |
| DELETE | /api/v1/auth/delete-account | Полное и безвозвратное удаление аккаунта текущего пользователя из базы данных |

||||
|-|-|-|
| GET    | /api/v1/user/:id/list       | Список проектов (досок) пользователя |
| GET    | /api/v1/board/:id           | Получение информации проекта из представления (view) board |
| POST   | /api/v1/projects            | Создание проекта |
| PUT    | /api/v1/projects/:id        | Обновление информаци проекта (name, description) |
| DELETE | /api/v1/projects/:id        | Удаление проекта |

||||
|-|-|-|
| POST   | /api/v1/columns             | Создание колонки |
| DELETE | /api/v1/columns/:id         | Удаление колонки |
| PATCH  | /api/v1/columns/:id/rename  | Переименование колонки |
| PUT    | /api/v1/columns/:id/move    | Перемещение колонки |
| POST   | /api/v1/columns/insert      | Вставка колонки слева или справа относительно текущей |
| DELETE | /api/v1/columns/:id/clear   | Удаление всех задач в текущей колонке |

||||
|-|-|-|
| POST   | /api/v1/tasks               | Создание задачи |
| PUT    | /api/v1/tasks/:id           | Редактирование задачи |
| DELETE | /api/v1/tasks/:id           | Удаление задачи |
| PUT    | /api/v1/tasks/move          | Перемещение задачи между колонками и вертикальное позициоирование в колонке |


## 💡 Функциональные возможности приложения
**Авторизация и Регистрация**: Полноценная аутентификация пользователей с валидацией форм.  
**Управление профилем**: Редактирование имени, email-а и смена пароля пользователя. Удаление профиля пользователя.  
**Отображение списка проектов**: Динамическая сетка карточек, доступная только авторизованным пользователям.  
**Создание, редактирование и удаление проектов**: Модальные окна для добавления новых, изменения данных (название, описание) и удаления досок.  
**Управление колонками через контекстное меню**: Создание, переимнование, линейное перемещение («Влево» / «Вправо»), точечная вставка («Добавить слева» / «Добавить справа»), очистка и удаление колонок.  
**Управление задачами внутри колонок**: Модальные окна, позволяющие создать, переписать контент задачи или удалить её.  
**Режим фокусировки**: Переключатель в шапке доски, который по клику полностью скрывает все элементы создания (пунктирные карточки «Новая задача» и «Добавить колонку»)  
**Изоляция кликов интерфейса**: Редактирования карточек проектов и задач исключает ложные срабатывания (пользователь может открыть редактирование проекта\задачи, не совершая при этом переход внутрь самой доски или перемещение задачи).  
**Производительное агрегирование данных**: Загрузка структуры всей доски (проекта, всех его колонок и всех вложенных задач) происходит за один SQL-запрос благодаря индексированному SQL-представлению VIEW board с агрегацией связей через json_agg и json_build_object.  
**Интерактивный Drag & Drop задач**: Свободное перемещение карточек задач как внутри одной колонки (изменение позиции), так и между разными колонками.  
*Синхронизация с БД*: Все данные сохраняются на сервере в СУБД PostgreSQL.
